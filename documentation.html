BUILDING SAKURA

Here we will guide you on how to build Project Sakura for your device.

HOW TO SETUP A BUILD ENVIRONMENT

To Build Project Sakura you can use any 64 Bit Linux Distro, though we recommend using Ubuntu 18.04, LinuxMint, MXLinux or Manjaro. You can setup the build environment using Akhil Narang's Script 
git clone https://github.com/akhilnarang/scripts; cd scripts; sudo bash setup/android_build_env.sh
Or You can install all the packages manually using the commands below. 
sudo apt-get install openjdk-8-jdk && sudo apt-get update && sudo apt-get install git-core gnupg flex bison gperf libsdl1.2-dev libesd0-dev squashfs-tools build-essential zip curl libncurses5-dev zlib1g-dev openjdk-8-jre openjdk-8-jdk pngcrush schedtool libxml2 libxml2-utils xsltproc lzop libc6-dev schedtool g++-multilib lib32z1-dev lib32ncurses5-dev lib32readline-gplv2-dev gcc-multilib maven tmux screen w3m ncftp

SYNCING SOURCES

To Sync ProjectSakura Sources you will need to execute the following command (PS: This command will shallow sync the sources which saves bandwidth and storage space if you are just going to build and not do any source side changes.) 
repo init --depth=1 -u git://github.com/ProjectSakura/android.git -b 11 && repo sync --current-branch --force-sync --no-clone-bundle --no-tags --optimized-fetch --prune -j `nproc`

BUILDING SAKURA

Now execute the below command to start building ProjectSakura, replace "devicecodename" with your device's code. E.g, for ASUS ZENFONE MAX PRO M1 we will write X00T. 
. build/envsetup.sh && lunch lineage_devicecodename-userdebug && mka bacon -jx

FLAGS PRESENT IN PROJECT SAKURA

In this article we will tell you about all the available Flags in ProjectSakura

LIST OF ALL FLAGS WITH THEIR FUNCTIONALITY.

SAKURA_BUILD_TYPE=coregapps //For building ProjectSakura with Builtin Core GAPPS
SAKURA_BUILD_TYPE=basicgapps //For building ProjectSakura with Builtin Basic GAPPS
SAKURA_BUILD_TYPE := microg //For building ProjectSakura with Builtin MICROG.
SAKURA_LAWNCHAIR := true //For building ProjectSakura with default Lawnchair Launcher.
TARGET_USES_BLUR := true //For building ProjectSakura with option to toggle Notification shade Blur.
EXTRA_FOD_ANIMATIONS := true //For building ProjectSakura with FODAnimations resources.

INSTALLING SAKURA

This article only cover universal steps that you need to take and things may differ for every device so it's better if you also follow the XDA thread for your device.

DOWNLOAD PROJECT SAKURA

To flash ProjectSakura you will need to get the flashable ROM zip for your device. We have a download center where you can get builds for every device that we support, Go to ProjectSakura | Downloads

If you cannot find the build for your device on our download center then it probably is unsupported by us. 
Note: Some devices also need specific firmwares and Vendors to be flashed with ROM. To know more you should check the XDA thread or Telegram Chats of your device.

FLASHING PROJECT SAKURA

After you have downloaded all the needed files, you can proceed with the flashing process. Boot your device into recovery mode and go to advanced wipe after that wipe dalvik, cache, system, vendor and data.

Once you have wiped, flash the required firmware / vendor zips and the flashable ROM zip. Then reboot to system and enjoy a clean Android experience. 

Note: ProjectSakura provides three types of builds GAPPS, VANILLA and MICROG. You may or maynot need to flash a separate GAPPS package depending on the type of the build; maintainer is releasing.

GAINING ROOT ACCESS

You will need to flash Magisk in order to gain root access. You can download the flashable magisk zip from it's Github Repo. We only recommend you to download magisk from verified sources.

REPORTING A BUG

Here we will guide you on how to report a bug related to Project Sakura

WHERE SHOULD I REPORT BUGS

You can report Bugs that are related to our Source on our Bug Reports Repo. If you are facing a bug related to a specific device then you can report about it to the device maintainer in our Telegram Chat or the device specific support Group.

WHAT DO I NEED TO INCLUDE IN THE BUG REPORT

If you are going to report the bug on our Bug report Repo then you can use the template that we have designed for opening Issues there. If you are going to report any device specific bug then you will need to write the description of the bug and properly explain the procedure of reproducing the bug. It is also required to attach screenshots or a logcat.

APPLY FOR PROJECT SAKURA MAINTAINERSHIP

In this article we will tell you how you can become an Official ProjectSakura Maintainer.

GUIDELINES

You don't need too many skills to be a device maintainer. Though you need to fullfill these simple guidelines. 
1. You must own the device. You can maintain the device that have minimal hardware difference and use common trees with your device. BLIND building is absolutely NOT allowed.

2. You must know Basic git ( creating pull requests, properly writing commits, cherry-picking and other basic stuff).

3. You must have made some contributions towards your device community instead of just taking a device tree and compiling.

4. You must know how to read basic logcats.

5. You must have to compile Unofficial builds before you apply for Official. Do NOT apply as soon as you built an unofficial build and instead maintain it for a period of 2 months. Applying before you have even built will lead to instant rejection and we will blacklist you in our database.

6. Co-Maintaining is allowed but you cannot be a shadow maintainer for some other guy.
You will recieve an email or Telegram message if we acknowledge your application. In some cases we may talk to you for some period of time to decide if we should accept or reject your application.
